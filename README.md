# Clawnitizer: AI Agent Session Optimizer 🧼🤖

An ultra-lightweight, single-file, zero-dependency visual interface written in PHP to inspect, prune, and sanitize **JSON Lines (JSONL)** chat logs generated by autonomous AI agents like `openclaw` and `picoclaw`. 

Think of it as a quick "hack & clean" scratchpad for your agent's memory. When building agent loops, models can easily fall into repetitive cycles, generate massive multi-kilobyte reasoning chains, or get bogged down by messy tool payload echoes. **Clawnitizer** lets you manually step in, trim out the off-topic turns, and keep your API contexts fast, lean, and highly cost-efficient.

## 🧠 Removing the Extra Baggage (Why Clawnitizer?)

* **The "Reasoning" Focus:** With the rise of deep reasoning models (like DeepSeek-R1), logs are flooded with massive `<thought>` or `reasoning_content` blocks. Once the final answer is achieved, these thoughts are dead weight. Wiping them out saves massive amounts of input tokens on subsequent turns.
* **Did Read, Did Realize:** Let's face it—sometimes a human expert looks at a log and says, *"This whole sequence is garbage, let me delete it."* Clawnitizer gives you the scalpel.
* **Hello, Hallucination (Human Edition):** Humans "hallucinate" too. You will open a log and realize, *"Why did I ask that question? I was completely off-topic."* Now you can retroactively erase your own rabbit holes.
* **Emotions Run Wild:** Did you get frustrated and type, *"Why aren't you working?! GPT-4o would have figured this out by now!"* inside your prompt? Emotions waste tokens. Clean up your raw prompts so the agent stays focused.
* **Less Garbage In, Less Garbage Out (Hopefully):** Look at why your agent failed or wasted 20,000 tokens, manually delete the bad turns, tweak a prompt, and hit **Save** to re-run the experiment clean.

> 💡 **Framework Note:** `picoclaw` saves sessions inside the `.picoclaw/workspace/sessions/` path, named exactly after your API key (e.g., `secret-API-key.jsonl`). The session file uses the **JSON Lines (JSONL)** format, which is the strict standard architecture used by OpenAI, Anthropic, and Mistral APIs for `user`, `assistant`, and `tool` tracing. OpenClaw stores its session files in the following directory:~/.openclaw/agents/agentId/sessions/. OpenClaw stores its data in the exact same format as PicoClaw. Always make a backup copy of both your meta.json & .jsonl file!

## ✨ Features

* 📦 **Single-File Architecture:** Just pull `index.php` into your container or local PHP development space. Zero `npm install`, zero composer dependencies, zero database configuration.
* 📈 **Live Token Optimization Ticker:** Displays an approximate token count per turn (~4 chars/token heuristic). Deleting a card instantly updates your total context footprint via vanilla JavaScript before you hit save.
* 👁️ **Visible Formatting Breaks:** Swaps standard invisible escape formatting (`\n`) with a clear `[NEWLINE]` anchor string inside editable fields so you never accidentally corrupt string structures.
* 🛠️ **Preserves Architecture Elements:** Caches complex internal parameters like `tool_calls` arrays and explicit `tool_call_id` variables using hidden state headers, ensuring updated logs seamlessly import back into your framework pipeline.
* 📥 **One-Click Markdown Transcripts:** Instantly output your clean, pruned conversation sessions into a structural `.md` log file—perfect for manual audit reviews or feeding into another LLM for context checkups.

## 🚀 Getting Started

### 1. Installation
Clone this repository or copy the `index.php` script into your local web root directory (e.g., WAMP, XAMPP, or LAMP setup).

### 2. Prepare Data Environment
Grab a copy of your session file (e.g., `YOUR_SECRET_API_KEY.jsonl` if agent is `picoclaw`) from your agent server, place it in the Clawnitizer folder, and rename it to `session.jsonl`.

**Example Windows 10 (WAMP) Structure:**
```bash
c:/wamp64/www/clawnitizer/
├── index.php
└── session.jsonl

```
### 3. Launch & Clean

Open your browser and navigate to `http://localhost/clawnitizer` to start cleaning your files.

### 4. Deploy It Back

Once you click **Save Optimized JSONL**, copy your newly trimmed `session.jsonl` file back into your agent's workspace directory. Don't forget to **rename** the file back to its **original filename**.

> 💻 **Cross-Platform Tip:** If your agent runs on a remote Linux server while you edit on Windows 10, use a quick `scp` command, WinSCP, or an SFTP client to pull/push the session file between machines!

## ⚠️ Important Safety & Experimental Notices

- 🔴 **CRITICAL RULE: SYSTEM CRASH PREVENTION:** Never run this editor on a live file that an active AI agent loop is currently writing to! Always make a backup copy of your original session file before hitting save to prevent file truncation or session locking conflicts.
    
- 🧪 **Experimental Sandbox Tool:** This script is primarily built for local diagnostics, troubleshooting, and developmental playground testing.
    
- 🧩 **State Interruption Risks:** Use logical judgment when trimming conversation elements. Erasing a `user` query while keeping its matching `tool` call response might compromise historical dependency structures and cause pipeline loops in your agent framework.
    
- 🔒 **Network Exposures:** This standalone script lacks embedded authentication or authorization checks. **Never expose this interface publicly onto shared host names or live production servers.** Keep its execution strictly limited to local loopback listeners (`localhost`, `127.0.0.1`).
